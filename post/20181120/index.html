区分 ArchiMate 中的 Composition 和 Aggregation

最近因为工作涉及描述企业架构（Enterprise Architecture, EA） ，研究了下 ArchiMate，
看到第5章 Relationships，发现其中的 Composition 和 Aggregation 非常 confuse。 官方描述如下：

a composition relationship is that the whole or part of the source element is composed of the whole of the target element.

an aggregation relationship is that the whole or part of the source element aggregates the whole of the target element.

整个定义区别就是 is composed of 和 aggregate 的区别，总之我没有看懂有什么区别。

aggregation官方样例：

网上找了下，中文资源有 周金根博客 ，现在回头来看已经可以看懂了，不过当时也没有完全理解。

我推荐Phil Curnow 的解释，其采用了程序员较容易理解的类的概念来解释这两者，我觉得非常赞。在此简单翻译记录。

Composition 理解为 “part-of”, 一个便于比较的例子如下：

上图中的 Car 和 Engine 抽象为两个类，Engine 的生命周期由 Car 来决定。

public class Engine
{
	//some code
}

public class Car

{
    Engine e = new Engine();
	//some code
}

Aggregation 理解为 “has-a”, 一个便于比较的例子如下：

上图也可以抽象为 Person Address 两个类，但是 Address 的生命周期 和 Person 无关。

public class Address
{
	\\some code
}

public class Person

{

     private Address address;

     public Person(Address address)

     {

         this.address = address;

     }

     \\some code
}

Address address = new Address();
Person person = new Person(address);

回过头来看官方用例，基本符合这个说法。
